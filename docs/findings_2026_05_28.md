## Program A — Economics-Authority Split (Findings F1–F5)

### F1 — Balance-only rescue still overwrites legacy entry economics (P1, highest ROI)

**Violation.** `chain_reconciliation` rescue path overwrites `entry_price`, `cost_basis_usd`, `size_usd`, `shares` with chain aggregate, even on **balance-only** rescues (no linked trade fact). This collides three distinct objects:
- decision/submission economics (limit, intended notional, submitted shares)
- verified fill economics (avg fill price, filled cost basis, shares filled)
- venue position/balance economics (chain avg price, chain cost basis, chain shares)

**Concrete failure.** Limit 0.38 submit → crash → chain balance avg 0.44. Rescue with `fill_authority=venue_position_observed` correctly degrades authority — but `entry_price` is silently overwritten to 0.44. Downstream P&L / fill-quality / training-explanation conflates chain-aggregate as fill-economics.

**Existing defenses insufficient.** `fill_authority` tells readers but cannot stop legacy field misread; `corrected_executable_economics_eligible` covers only D6-corrected rows.

**New invariant.**
```
LocalIntentEconomics       = submitted_limit_price, submitted_notional_usd, submitted_shares
VenueTradeFillEconomics    = avg_fill_price, filled_cost_basis_usd, shares_filled, venue_trade_fact_id
VenuePositionObservedEcon  = chain_avg_price, chain_cost_basis_usd, chain_shares, venue_position_observed_at
EffectiveExposure          = derived; source_authority = venue_trade_fill | venue_position_observed
```

**Files to change.**
- `src/state/portfolio.py` — add chain economics fields if missing; classify legacy fields as compatibility/derived.
- `src/state/chain_reconciliation.py` — balance-only rescue writes chain economics fields, **not** entry/fill fields.
- `src/engine/lifecycle_events.py` — `build_venue_position_observed_canonical_write` payload carries `chain_avg_price`/`chain_cost_basis_usd`/`chain_shares`.
- `src/state/db.py` — `position_current` schema adds chain economics cols; status views compute effective exposure from authority-specific fields.
- `src/execution/exit_triggers.py`, `src/engine/monitor_refresh.py` — consume effective-exposure object, not raw `Position.shares`.

**Delete/shrink.** `cost_basis_chain_mutation_blocked_total` branches around fields chain-balance must no longer mutate. Comments saying "chain overrides entry price".

**Acceptance tests.**
1. Balance-only rescue preserves submitted limit + submitted notional.
2. Balance-only rescue sets chain economics fields (not entry fields).
3. Trade-verified fill sets fill economics.
4. No training/reporting path can use chain economics as verified fill economics.

### F2 — Event timestamps collapse decision/fill/observation time (P1)

**Violation.** `occurred_at` and `projection_updated_at` chosen via fallback chains and string `max()` — mixing decision-time, submit-time, fill-time, venue-observed-time, reconcile-time.

**Evidence.**
- `projection_updated_at()` uses string `max()` over raw ISO; lexicographic ≠ chronological with mixed `Z` vs `+00:00`.
- `build_venue_position_observed_canonical_write.occurred_at = _non_empty(entered_at, chain_verified_at, projection["updated_at"])` — legacy entered_at wins despite event being a later venue observation.
- `REVIEW_REQUIRED.occurred_at` similar fallback chain.

**Concrete failure.** Legacy pending row has old `entered_at`. Chain observation today. Event timestamp picks old `entered_at`. Replay / settlement window / training audit mis-orders recovery.

**New invariant.** Every canonical event builder receives explicit `event_observed_at`. `VENUE_POSITION_OBSERVED.occurred_at = venue_position_observed_at`. `REVIEW_REQUIRED.occurred_at = review_detected_at`. `entered_at` only used as event time when event IS the entry fill.

**Files.** `src/engine/lifecycle_events.py` (explicit ts args), `src/state/chain_reconciliation.py` (pass `now` as `venue_observed_at`/`review_detected_at`), `src/state/portfolio.py` (add `entered_at_authority` if reconstruction needed).

**Delete.** `_non_empty(..., entered_at, chain_verified_at, updated_at)` ordering patterns; string `max()` ts selection; replace with parse-and-normalize helper.

**Acceptance tests.**
1. Legacy row old entered_at + new chain observation → `VENUE_POSITION_OBSERVED.occurred_at == chain_observation_time`.
2. Mixed `Z` + `+00:00` timestamps sort chronologically after parse.
3. Fabricated `entered_at` cannot become fill time / venue observation time without authority label.

### F3 — Legacy NULL fill_authority rows fail closed without deterministic backfill (P1/P2)

**Violation.** Migration added `fill_authority` as nullable additive column; legacy rows stay NULL until "future projection rewrite". Harvester gates training on NULL — fails closed. Verified historical examples silently excluded.

**New invariant.** Every row has authority classification after migration:
- `venue_confirmed_full`
- `venue_confirmed_partial`
- `venue_position_observed`
- `legacy_unknown`

**Files.** `src/state/ledger.py` or migration script (deterministic backfill rules), `src/execution/harvester.py` (distinguish `legacy_unknown` from NULL/unmigrated), `src/state/db.py` (report query for unclassified rows).

**Backfill rules.**
- Linked positive `venue_trade_facts` → `venue_confirmed_full` (or `partial` per row count).
- `VENUE_POSITION_OBSERVED` event present → `venue_position_observed`.
- Positive fill payload with verified causality → verified.
- Otherwise → `legacy_unknown`.

**Acceptance tests.**
1. Legacy verified row with linked trade fact backfills to `venue_confirmed_full`.
2. Legacy balance-only event backfills to `venue_position_observed`.
3. Ambiguous row → `legacy_unknown`.
4. Harvester gate distinguishes `legacy_unknown` from unmigrated NULL.

### F4 — Runtime phase still derived from mutable strings (P2 invasive)

**Violation.** `canonical_phase_for_position()` still calls `phase_for_runtime_position(state, exit_state, chain_state)`. Mutable runtime strings (`voided`, `settled`, `quarantined`, `pending_exit`, `pending_tracked`, `entered`, `holding`) are projection authority.

**New invariant.** Event builders take explicit `phase_before`, `phase_after`. `Position.state` derived **after** projection, never inputs to phase decision.

**Files.** `src/engine/lifecycle_events.py` (require `phase_after`), `src/state/chain_reconciliation.py` (typed event constructors not mutate-then-build), `src/state/portfolio.py` (module doc → projection-cache).

**Delete/shrink.** `phase_for_runtime_position` as money-path authority — keep only as legacy adapter.

**Acceptance tests.**
1. Direct mutation of `Position.state` cannot change canonical projection unless event builder receives a legal phase transition.
2. Static test forbids `build_position_current_projection(position)` deriving phase via runtime strings in new code.

## Program B — Generation-Naming Eradication

### Target Rule

Repo-owned current source tree must have **zero**:
- `version`, `_v1`, `_v2`, `v1_`, `v2_`, `vnext`
- `schema_version`, `event_version`, `model_version`, `calibration_version`, `signal_version`, `data_version`, `pricing_semantics_id`, `execution_cost_basis_version`
- `legacy_archived`, `*_old`, `*_new`, `*_legacy`

Exceptions:
- Third-party dependency metadata (Python package output).
- Git history (not scanned).

### Replacement Model

| Old | New |
|-----|-----|
| `SCHEMA_VERSION`, `schema_version` col, `PRAGMA user_version` | schema fingerprint (`tests/state/_schema_fingerprint.txt`, `scripts/check_schema_fingerprint.py`) |
| `_v1`, `_v2`, `vnext` table/file/function | canonical base name |
| `event_version` | drop if always 1 |
| `model_version`, `calibration_version` | `model_artifact_id`, `calibration_model_id` |
| `signal_version` | `signal_id` |
| `pricing_semantics_id` | `pricing_semantics_id` |
| `execution_cost_basis_version` | `cost_basis_policy_id` |
| `data_version` | `dataset_id` / `source_snapshot_id` |
| `legacy_archived` | deleted, or `archive` if genuinely retained |
