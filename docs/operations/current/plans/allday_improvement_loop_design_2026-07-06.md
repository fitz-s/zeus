## 0. 目标(不变)

结算后实际资本利得(ground-truth graded)> 系统健康 > 代码库清晰度。非目标(loop 永不做):部署、动风险姿态、写 live DB、给自己升权。

## 1. 形态:两层

| 层 | 触发 | 职责 |
|---|---|---|
| **L1 tick** | launchd 每小时 :17 | 队列取**一**项 → 子代理执行 → 独立 verifier → journal。开头先查:无新证据且队列空 → 记一行空 tick 立即退出(证据驱动,不烧空转) |
| **L2 日析** | launchd 每日 UTC ~12:10(结算窗后) | settlement_skill_attribution join(cursor = journal 里上次 rowid)→ 账本升降级 → 队列重排 → 晨报 |

Lead 都是 opus;顶级档不用。子代理拓扑沿用全局法(scout=haiku 枚举、builder/analyst=sonnet、verifier=全新 context 证伪 brief,money-path 用 opus verifier)。

## 2. 状态:一目录三文件

```
zeus/loop/
  HALT           # 存在即全停(操作员 touch;唯一开关)
  JOURNAL.md     # append-only。tick 记录、ESCALATION: 行、DEVIATION: 行、VIOLATION: 行、cursor 值
  LEDGER.yaml    # 假设账本 + 改进队列合一(队列项就是账本条目的 next_action)
  prompts/{l1,l2}.md  bin/tick.sh  logs/
```

没有 digest、escalations、deviations、budget 文件。晨报从 JOURNAL grep 生成,不另存状态。假设账本法则:claim 可判伪、统计性结论 min_n=30、refuted 条目不删(防重提)、记忆/直觉进来一律 open + 零证据;**查询窗口预登记**(先写 probe 再看数);探索性切片发现不升 confirmed,必须新窗口预登记复测。机制性发现任何 n 可行动,但**机制性仅限 invariant 违反的纯代码证明 —— 凡涉 EV/校准/选市/sizing/市场行为一律统计性**。

## 3. 权限:三级,wrapper 机制强制(非 prose)

| 级 | 范围(按路径,机器可判) | 规则 |
|---|---|---|
| **AUTO** | `loop/**`、`docs/**`、`architecture/*.yaml`(registry 修复)、`tests/**`;`scripts/**` 仅限 manifest 无 dangerous_if_run/write_targets 且无 plist/manifest 引用者 | 分支 + verifier 绿 + boot smoke → 自主合并;单 commit 原子;**diff >20 文件或 >600 行即拒 + ESCALATION** |
| **PREPARE** | `src/**`(money path);一切动态入口(registry-loaded/string-dispatch)死码删除 | worktree + TDD + opus verifier → **永不合并**,diff 排队晨报 |
| **NEVER** | `config/**`、live DB、LaunchAgents、deploy、script_manifest 标危脚本 | 提案文字 only,不开 worktree |

证据查询一律 SQLite `mode=ro` URI;manifest 标 dangerous_if_run/write_targets 的脚本 loop 永不执行。

**强制机制(采纳 consult BLOCKER-1,拒收其 constitution/hash-pin 方案):** `tick.sh` 在 claude 退出后跑后置校验 —— 主树 diff 文件清单对照 AUTO allowlist,越界文件一律硬恢复到 HEAD 版本(worktree/分支不受影响)+ JOURNAL 记 `VIOLATION:` 行。prose 承诺不了的,wrapper 几行 sh 承诺。L2 晨报里 VIOLATION 非零 = 头条。

```sh
# tick.sh 骨架(全 sh;launchd 直调 binary,绕开 zsh function 的 proxy 注入)
[ -f "$LOOP/HALT" ] && exit 0
exec 9>"$LOOP/.lock"; flock -n 9 || exit 0
claude --model opus -p "$(cat $LOOP/prompts/$1.md)" --max-turns $MAX_TURNS ...
# 后置:allowlist 外的主树改动 → 恢复 HEAD 版本 + VIOLATION 行(机制,非约定)
```

停机:HALT / flock(不叠加)/ max-turns / 外层 process-group timeout + 日志字节帽(tick.sh 几行 sh)/ 同一队列项 3 连败标 blocked 升级。没有独立预算系统 —— max-turns × launchd 间隔就是预算。

## 5b. 官方 loop 规范对照(Anthropic《Getting started with loops》2026-07-06, @delba_oliveira)

官方口径与本设计**强对齐**:小模型跑量/强模型判断(=L1 sonnet / L2 opus)、auto 只给安全确定动作(=AUTO 层仅 docs/tests/死码,money-path 一律 PREPARE)、dynamic workflow 对抗式(=Workflow 扇出 + 对抗 verifier)、把失败编码回 harness 而非孤立修 bug(=antibody 测试 + 表示层契约 + wrapper 硬还原)、干净代码库是前提(=R0-R8 债务清算)。官方明言**不规定形式化 loop schema,系统设计压过模型 IQ** —— 背书操作员"最小机器"直令。**三条官方强调、本设计 underweight 的增量(启用前吸收,加法非返工):**
1. **Token 预算 lever(官方:poll 间隔匹配状态变化速度 + 监控每 agent 花费 + 小切片试点再放量)。** 本设计只有时间墙 + flock,无 token 预算/`/usage` 复核。**补:** tick.sh 加每 tick output-token 上限(超即 ESCALATION),L2 晨报报本周期累计 spend;首启用小切片试点(见下 checklist)。
2. **每 tick 可验证退出(官方 `/goal`:done 写 boolean/numeric + turn cap,模糊标准=烧 token)。** 程序级判据(settlement 后 EV)是数值的,但 tick 级 ACTION 是开放式"取一 packet"。**补:** L1 prompt 的 ACTION 段要求每 tick 声明本 tick 的可判 done(测试绿数/死码行删/证据 query 返回),达成或 max-turns 即止。
3. **小切片试点先于全量 proactive(官方 guardrail)。** 本启用序直接两条 launchctl 上线。**补:** 启用序前置一步 —— 先 HALT 存在下手动跑一次 tick.sh(dry 观察 JOURNAL + VIOLATION 行),确认后再移除 HALT。
