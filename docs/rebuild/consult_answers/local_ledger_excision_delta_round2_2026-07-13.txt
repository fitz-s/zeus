(b) Is “the pass survives; its economics writes die at LX-3R” sufficient?

Necessary, but not sufficient.

It is sufficient only when “the pass survives” means:

The pass has already been rewritten as an observer/resolver that persists causative facts and local protocol state, with all decisions and selectors detached from the old economics.

It is not sufficient if it means:

Leave the current code intact and conditionally skip or database-reject the final UPDATE.

Four conditions must hold at every named site before activation.

1. Causal fact completeness

Everything needed to reproduce the removed economics must already exist as immutable facts. A position event saying “projection repaired” is not enough if the exact fill set, fee set, source observation, or payout input is absent.

2. Reader and selector migration

A pass that no longer writes shares but still chooses candidates using stale shares remains coupled to the ledger.

This is especially acute in exchange_reconcile: the recovered ghost-sell selector searches position_current open phases and local shares before reconstructing the command. That lookup must move to canonical position identity plus attributed exposure. GitHub

3. Transaction separation

KEEP evidence must commit even when the old projection is forbidden. This requires code refactoring before activation, because a trigger aborts the statement’s transaction or savepoint.

4. Replacement publication or direct-fact liveness

Readers that need immediate safety state—especially duplicate-exit prevention—must either see a synchronously published reducer generation or be able to read the source command/trade facts directly.

The site-specific calls are:

position_duplicate_consolidator.py:186/369: special handling required. Replace the ongoing economic merge with identity supersession. Run or backfill this before constructing the new read model, otherwise duplicate identities can be double-counted.

command_recovery.py:2461/6215/8282/8387/8618/8683: special handling required as one migration packet. Preserve command terminalization, no-fill proofs, chain-absence observations, reviews, and unknown-side-effect handling. Remove direct economic repairs. Split savepoints and migrate candidate predicates. For each path, prove that the event/fact payload contains all source evidence needed by the reducer; an after-state copied from position_current is not sufficient.

exit_lifecycle.py:4680: special handling required because exits must continue. The exit fill fact and command terminal state must become sufficient to suppress another sell before the materialized read model catches up.

exchange_reconcile.py:1418: special handling required. Preserve reconstruction of the missing EXIT command and order evidence. Do not book P&L from the order price. Keep the position conservatively pending or partially exited until exact trade facts arrive.

exchange_reconcile.py:1813: special handling required. Preserve the external-close observation/operator disposition and let exact token truth plus attribution derive the result.

edli_position_bridge.py:1003: special handling required before LX-3R, not at LX-3R. It must first become a permanent fact bridge. Turning it off at the activation instant without that replacement can strand new EDLI fills.

projection.py:658: ordinary authority replacement. Stop its legacy economics insertion at activation; it may create a stable identity shell only if the target design still requires one. Economics belong in the reducer-owned read model.

The superior mechanism is therefore not eleven local feature flags. It is:

dual-capable fact-first code;

one truth epoch;

process capability fencing;

a database-level forbidden-column guard;

a generated static and runtime writer manifest.

Local flags alone are vulnerable to an omitted writer or stale daemon. A trigger alone is vulnerable to rolling back KEEP evidence. Both layers are required.
