### Summary (KM-C only)
- Mechanism count: 15 distinct mechanisms across 7 files.
- Buckets: B1=8, B2=5, B3=0, B4=2.
- **Hardest reshape**: command_recovery.py's `_decision_certificate_is_quarantined` (757-763, 3042-3118) is the SAME duplicated-checker problem flagged in KM-A for executor.py — two independent copies of certificate-revocation logic in the two live money-path callers. This ledger section adds confirmation: command_recovery.py's copy is byte-for-byte structurally identical to executor.py's (same import pattern, same reason-code check, same schema-qualifier lookup), which makes consolidation mechanically easy but still requires both call sites to be re-pointed in the same commit to avoid a window where they check different logic.
- **Sequencing risk flagged**: control_plane.py's `acknowledge_quarantine_clear` operator command (T6) is currently the ONLY release valve for stuck quarantined positions. Doc's own ordering (T6 "rides T2/T5") means T6 execution must not delete the ack mechanism until T5's replacement state has an equivalent operator release path wired — otherwise operators lose the ability to clear disputed positions mid-migration.
- **Family confirmed, not duplicated**: the 3 command_recovery.py repair functions (`_void_absorbed_chain_only_projection`, `repair_confirmed_phantom_voids`, `repair_confirmed_chain_absence_positive_projections`) plus chain_reconciliation.py's live-cycle equivalent (KM-A) all import the SAME constants (`ENTRY_AUTHORITY_CHAIN_ABSENCE_CHAIN_STATE` etc.) from chain_reconciliation.py — this is one coherent mechanism family split across recovery-time and live-cycle triggers, not independent reimplementation. Good news for the reshape: rename the shared constants once, all 4 call sites follow automatically.
- **New finding**: price_channel_ingest.py:1685-1710 is a 4th independent copy of the "quarantined+voided w/ positive chain_shares still counts as exposed" predicate (after cycle_runtime.py, portfolio.py x1 from KM-B, and canonical_monitor_position_rows from KM-A) — strengthens the consolidation recommendation from KM-A/KM-B.
