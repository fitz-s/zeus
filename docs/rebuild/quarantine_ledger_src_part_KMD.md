## Hardest reshape call
`src/data/replacement_forecast_calibration_quarantine.py`: the file's *name* screams disease but the actual code has zero quarantine shape — no side-table, no persisted exclusion, no global gate, just a stateless request→ALLOWED/BLOCKED classifier with reason codes (exactly T2's "legitimate" definition). The genuinely hard call is that grep finds only its own test file as a caller — no src/scripts caller anywhere (evaluator.py, event_reactor_adapter.py, calibration pipeline all clean of it), matching a prior investigation doc's finding that it's "dormant on the live path." That makes it tempting to call B3 dead-code-delete. I kept it B2 (rename, keep) because B3 in this ledger's definition is reserved for disease-scaffolding (helpers/tests that only exist to support minting/gating quarantine state) — this function's logic is real, correctly-shaped validation, just unwired; deleting live logic for lack of a caller is a separate dead-code question outside the quarantine-excision mandate, and total extermination only requires the WORD and the DISEASE PATTERN to hit zero, not the removal of every unused-but-correct validator.

The second-hardest call was the `settlements.authority='QUARANTINED'` mechanism (harvester.py + harvester_truth_writer.py, duplicated write path): it is NOT named in T1-T7 yet clearly matches T2's disease/legitimate split test almost exactly — scoped to one row, corrected by an evidence-gated reactivation trigger (`reactivated_by` provenance key), self-heals via INSERT OR REPLACE. Classified B2 standalone rather than folding it into an existing T-number, but flagged that both writer copies must rename in the same packet since they share one CHECK constraint and one trigger.
