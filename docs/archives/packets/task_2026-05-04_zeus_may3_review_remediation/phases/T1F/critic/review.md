## Caveats

- **C-1 (LOW)**: `T1F-COUNTER-EMITTED` invariant says counter is "incremented exactly once per blocked attempt and persisted in the project's existing counter sink." Implementation uses `logger.warning("telemetry_counter event=...")` which is the project's existing counter-via-log-tap convention, but `tests/test_polymarket_adapter_submit_safety.py` does NOT use a `caplog` fixture or otherwise assert the warning was emitted. The invariant is structurally guaranteed by code-path uniqueness (only paths returning the rejection codes are immediately preceded by the emit), but it is not test-asserted. Realist Check: detection of a future regression (someone removes the emit) would happen via log-volume monitoring in production, not unit test. Severity LOW because the structural guarantee holds. Not blocking; flag for security-reviewer to decide whether to require a `caplog` assertion.
- **C-2 (LOW)**: `src/contracts/venue_submission_envelope.py` is in `phase.json::files_touched` but actually unchanged in the worktree. Consistent with `scope.yaml` line 48-49 (in_scope ONLY if assertion-method validation gaps are found); line 107 was already complete. Documentation drift in phase.json, not a correctness issue.
