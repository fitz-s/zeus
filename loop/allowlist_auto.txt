# AUTO-tier path allowlist for loop/tick.sh (L1) and loop/daily.sh (L2).
#
# One glob per line; '#' starts a comment; blank lines ignored. Matched by
# scripts/ops/loop_guard.py::path_allowed against every path that changed
# DURING a tick (relative to the pre-tick snapshot, never against files the
# operator already had dirty). Any changed path that matches none of these
# is hard-restored to HEAD and logged as VIOLATION in loop/JOURNAL.md.
#
# `**` matches any depth (including zero extra segments); a bare `*` or `?`
# matches within one path segment only (does not cross '/') — this is a
# glob translator local to loop_guard.py, not stdlib fnmatch.
#
# Source of truth for scope:
#   docs/operations/current/plans/allday_improvement_loop_design_2026-07-06.md §3
#   (AUTO row) + docs/rebuild/EXECUTION_MASTER_2026-07-07.md §C2.
#
# WHO WRITES THIS FILE: only a human/operator commit. A tick can NEVER
# widen its own scope by editing this file (or loop/tick.sh, loop/daily.sh,
# loop/prompts/**) even though those paths sit under loop/** — they are
# GUARD_IMMUTABLE in scripts/ops/loop_guard.py::cmd_enforce, checked
# unconditionally before the allowlist itself. Any tick-made change to any
# of them is always a VIOLATION and gets hard-restored, full stop.
#
# THE LOAD-BEARING PROTECTION IS ELSEWHERE: enforce never reads this live
# file, and never reads any frozen copy kept inside the repo (an earlier
# version did — loop/.pre_tick_allowlist.{l1,l2} — and it was a real escape:
# that path sits under loop/**, gitignored but still repo-relative, so a
# tick's Bash tool could `echo 'src/**' >> loop/.pre_tick_allowlist.l1`
# mid-run and enforce would trust the tampered copy). tick.sh/daily.sh now
# `cp` this file into a per-invocation `mktemp -d` directory OUTSIDE the
# repo tree entirely, BEFORE claude runs, and pass THAT path to
# `enforce --allowlist-snapshot`. The GUARD_IMMUTABLE check above is
# belt-and-suspenders; the out-of-repo snapshot is what actually defeats a
# tick guessing a plausible REPO-RELATIVE path for the trusted artifact.
# It does NOT defeat active $TMPDIR enumeration by the tick's own Bash tool
# (same OS user, no process isolation) — see loop/tick.sh's SAFETY section
# KNOWN LIMITATION note for that accepted residual.
#
# WHO READS IT: scripts/ops/loop_guard.py enforce, via the out-of-repo
# snapshot, every tick. If the live file goes missing at snapshot time, the
# `cp` in tick.sh/daily.sh fails under `set -e` and the tick aborts BEFORE
# claude ever runs (fail closed). If the snapshot somehow still fails to
# load inside enforce, load_allowlist's OSError is caught and treated as
# zero patterns — allows nothing, so every new-this-tick change becomes a
# violation (fail closed there too). Do not let this file go stale
# regardless; a permanently-empty-patterns enforce quarantines every
# legitimate AUTO change too, which is safe but makes the loop useless.

loop/**
docs/**
tests/**
architecture/*.yaml

# scripts/** is intentionally NOT allowlisted here. The design's AUTO row
# only admits scripts/** entries with no dangerous_if_run/write_targets in
# architecture/script_manifest.yaml and no plist/manifest reference — that
# is a manifest-aware judgment call, not something a path glob can express,
# and the design's §5 "拒收" list explicitly rejects building a machine
# classifier for this. Until such a check exists, ANY scripts/** change made
# by a tick is out of AUTO scope and gets quarantined; script fixes route
# through PREPARE (queued diff, operator merges).
